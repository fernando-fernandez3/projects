# Contributing to GitFam

Thank you for your interest in contributing to GitFam! This project aims to help families preserve their legacy for future generations.

## Two Types of Contributions

### 1. Contributing to the GitFam Framework
Improvements to the templates, tools, and documentation that help all families.
This should be general enough to benefit a wide range of users, otherwise consider using your fork for personal use.

### 2. Using GitFam for Your Family
Fork this repository and create your own family archive.

---

## Contributing to the Framework

### What I'm Looking For

**Documentation**
- Clearer setup instructions
- Video tutorials
- Translations to other languages
- FAQ sections

**Templates**
- Better interview questions
- Profile templates for different cultures
- Templates for specific types of documents

**Tools & Scripts**
- Automation scripts
- Family tree visualizers
- Video transcription helpers
- Backup automation
- Web interfaces for non-technical users

**Examples**
- Sample family structures (anonymized)
- Best practices from real usage
- Creative solutions to common problems

### How to Contribute

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
4. **Test thoroughly**
5. **Commit with clear messages**
   ```bash
   git commit -m "Add: New interview question template for military service"
   ```
6. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```
7. **Open a Pull Request**

### Code Style

- **Scripts:** Use bash for cross-platform compatibility, include comments
- **Documentation:** Use Markdown, keep language clear and welcoming
- **Templates:** Include examples and comments explaining each section

### Commit Message Format

- `Add:` New features or files
- `Fix:` Bug fixes
- `Update:` Changes to existing content
- `Docs:` Documentation changes
- `Refactor:` Code restructuring

## Using GitFam for Your Family

### Getting Started

1. **Fork this repository**
   - Click "Fork" at the top of the GitHub page
   - This creates your own copy you can customize

2. **Rename your fork**
   - Go to Settings ‚Üí Repository name
   - Rename to something like "smith-family-history"

3. **Make it private** (recommended)
   - Go to Settings ‚Üí Visibility
   - Change to Private (if you want privacy)

4. **Start adding your family**
   - Follow the [Getting Started guide](GETTING_STARTED.md)
   - Use the helper scripts in `/tools`

### Sharing Your Experience

While your family content should stay private, I'd love to hear:
- What worked well
- What was confusing
- What features you wish existed
- Creative solutions you came up with

Share via:
- GitHub Discussions (anonymized)
- Issues (for bugs or feature requests)
- Pull requests (for framework improvements)

## Feature Requests

Have an idea for improving GitFam? We want to hear it!

**Before requesting:**
- Check if it already exists in Issues
- Think about how it helps ALL families (not just yours)

**When requesting:**
- Explain the problem you're solving
- Describe your proposed solution
- Share any examples or mockups
- Explain how this helps preserve family history

---

## Bug Reports

Found something broken?

**Include:**
- What you were trying to do
- What you expected to happen
- What actually happened
- Your operating system
- Git version (`git --version`)
- Git LFS version (`git lfs version`)

## Community Guidelines

GitFam is about honoring family and preserving legacy. We expect:

- **Respect** - This is about families, keep it positive
- **Privacy** - Never share others' family content without permission
- **Helpfulness** - We're all learning together
- **Patience** - Contributors are volunteers with their own families
